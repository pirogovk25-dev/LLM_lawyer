RAG-пайплайн для анализа юридических документов

Проект представляет собой систему Retrieval-Augmented Generation (RAG), предназначенную для автоматизированного формирования ответов на юридические вопросы на основе загруженной базы нормативных правовых актов. В качестве векторной базы данных используется Qdrant с применением гибридного поиска и алгоритма Reciprocal Rank Fusion (RRF). Управление логикой агента реализовано через LangGraph, а генерация ответов осуществляется локальной языковой моделью Qwen 2.5 (7B) посредством Ollama. Оценка качества работы пайплайна проводится с использованием фреймворка Ragas.

Структура проекта

Весь пайплайн реализован в едином файле LLM.ipynb. Для удобства навигации и проверки код логически разделен на этапы с помощью заголовков Markdown. 

На первом этапе (Чанкирование) производится чтение исходных документов в формате .docx, их очистка от технических артефактов и разделение на смысловые фрагменты с помощью RecursiveCharacterTextSplitter. Исходные документы должны располагаться в директории ./data. На втором этапе (Ground Truth) формируется эталонный набор данных, содержащий вопросы, релевантные контексты и ожидаемые ответы для последующего тестирования. Третий этап (Реализация поиска) включает векторизацию полученных фрагментов моделью BAAI/bge-m3, формирование плотных и разреженных векторов, а также реализацию функции гибридного поиска в Qdrant. Заключительный этап (Оценка) посвящен сборке графа агента в LangGraph и автоматическому тестированию сгенерированных ответов метриками Ragas.

Инструкция по запуску

Для запуска проекта необходимо установить требуемые зависимости которые можно найти в файле requirements.txt

Векторная база данных Qdrant разворачивается локально через Docker с помощью команды docker run -p. Локальная языковая модель Qwen запускается через платформу Ollama командой ollama run qwen2.5:7b. После успешного запуска базы данных и модели следует открыть файл LLM.ipynb в среде Jupyter Notebook и выполнить ячейки последовательно.

Оценка качества и расчет метрик

При тестировании системы фреймворком Ragas рассчитываются три основные метрики: faithfulness, answer_relevancy и answer_correctness. Результаты тестирования сохраняются в файл rag_evaluation_final_report.csv.
