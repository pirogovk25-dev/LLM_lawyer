RAG-пайплайн для анализа юридических документов

Проект представляет собой модульную систему Retrieval-Augmented Generation (RAG), предназначенную для автоматизированного формирования ответов на юридические вопросы на основе загруженной базы нормативных правовых актов. В качестве векторной базы данных используется Qdrant с применением гибридного поиска и алгоритма Reciprocal Rank Fusion (RRF). Управление логикой агента реализовано через LangGraph, а генерация ответов осуществляется локальной языковой моделью Qwen 2.5 (7B) посредством Ollama. Оценка качества работы пайплайна проводится с использованием фреймворка Ragas.

Структура проекта

Весь пайплайн разделен на несколько функциональных модулей для обеспечения стабильной работы и эффективного управления памятью.

В блокноте 1_data_prep.ipynb производится чтение исходных документов в формате .docx, их очистка от технических артефактов и разделение на смысловые фрагменты с помощью RecursiveCharacterTextSplitter. Исходные документы располагаются в директории ./data. Блокнот 2_indexing.ipynb отвечает за векторизацию полученных фрагментов моделью BAAI/bge-m3, формирование плотных и разреженных векторов, а также создание коллекции в Qdrant. Основная логика работы системы сосредоточена в файле app/main.py, где развернут бэкенд на FastAPI и собран граф агента в LangGraph для обработки запросов. Заключительный этап тестирования вынесен в блокнот 3_evaluation.ipynb, который включает расчет метрик Hit Rate и MRR, а также автоматическую проверку ответов фреймворком Ragas.

Инструкция по запуску

Для запуска проекта необходимо установить требуемые зависимости из файла requirements.txt.

Векторная база данных Qdrant разворачивается через Docker Compose или запуском локального исполняемого файла. Локальная языковая модель Qwen запускается через платформу Ollama командой ollama run qwen2.5:7b. Работа начинается с последовательного выполнения ячеек в ноутбуках 1_data_prep.ipynb и 2_indexing.ipynb для подготовки и загрузки базы знаний. После завершения индексации запускается основной сервер командой python app/main.py, после чего API становится доступно по локальному адресу.

Оценка качества и расчет метрик

При тестировании системы в блокноте 3_evaluation.ipynb рассчитываются показатели точности поиска и генерации. Фреймворк Ragas вычисляет три основные метрики: faithfulness, answer_relevancy и answer_correctness. Результаты тестирования и подробная статистика сохраняются в файл rag_evaluation_final_report.csv
